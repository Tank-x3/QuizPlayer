# アップデートメモ

## 1. カテゴリごとにクイズをまとめてアコーディオン表示
- 現在、問題データに記述されているカテゴリ情報をもとに、自動的にまとめて表示までは実装されているが、クイズの総数が増えて視認性・一覧性が悪くなっている
- そこで、カテゴリ単位でアコーディオン表示を行い、ユーザー（プレイヤー）が見たいカテゴリだけを展開できるようにする
- アコーディオンの開閉状態は記録しない。ページをリロードしたら、デフォルトですべて閉じた状態に戻す

## 2. カテゴリの並び替え
- 現在、カテゴリの並び順が制御できる仕様になっていない
  - これにより、関連性のある複数のカテゴリが離れて表示されてしまうことがある
- 管理者（私）が、カテゴリの並び順を自由に変更できるようにする
- 並び替えたあとの順番をファイルに保存し、次回以降の表示に反映させる

## 3. 問題データと管理用データの整合性チェック
- 1/2の各項目とも関連するが、問題データと管理用データの整合性をチェックする仕組みを導入する
  - `src\data\quiz_list.json`と問題本体のデータが一致しているかチェック（リストにない問題ファイルや、リストに記載があるのに存在しない問題ファイルがないか）
  - 問題データに記述されているカテゴリ情報と、管理用データ（並び順制御、新設する？）のカテゴリ情報が一致しているかチェックする
    - 一致していない場合は、エラーメッセージを表示する

## 4. 問題メーカー/整合性チェッカーをアプリケーション化する？
- 現在、問題メーカーはhtmlファイルになっているが、ファイルの保存操作に手間がかかる
- ファイルの自動保存が可能なアプリケーションにすると、管理が楽になるのではないか？
- 整合性チェッカーも複数ファイルの読み取りと、（問題がある場合）更新の必要があるため、同様にアプリケーション化する？
